Nostrbox Email Login Flow

Three phases: Registration → Magic Link Login → Key Self-Hosting

Phase 1 Registration

Client generates keys, encrypts with password, server stores encrypted blob

sequenceDiagram
    participant U as 👤 User
    participant W as 🌐 Web App
    participant S as ⚙️ Server

    U->>W: Click "Register with Email"
    W->>W: Generate Nostr keypair (nsec / npub)
    U->>W: Enter password + email address
    W->>W: Encrypt nsec → ncryptsec (NIP-49)
    Note over W: Password never leaves browser
    W->>S: email.register { npub, ncryptsec, email }
    S->>S: Store email_identity record
    S->>S: Create registration request
    S-->>W: ✓ Registered (pending approval)
    Note over S: Admin reviews and approves

Phase 2 Magic Link Login

Server sends single-use token via email, client redeems and decrypts keys locally

sequenceDiagram
    participant U as 👤 User
    participant W as 🌐 Web App
    participant S as ⚙️ Server
    participant E as 📧 Email

    U->>W: Enter email → "Send login link"
    W->>S: email.login { email }
    S->>S: Generate token (256-bit, 15 min TTL)
    S->>E: Send magic link via Resend
    S-->>W: ✓ Check your email
    Note over S: Same response whether
email exists or not

    U->>E: Open inbox → click magic link
    E->>W: Redirect with token
    W->>S: email.redeem { token }
    S->>S: Validate & consume token
    S-->>W: Return ncryptsec + npub

    U->>W: Enter password
    W->>W: Decrypt ncryptsec → nsec
    W->>W: Store nsec in sessionStorage
    Note over W: Signs all Nostr events:
NIP-42 auth, operations, etc.

Phase 3 Go Sovereign

User exports keys to NIP-07 extension or Amber, server deletes encrypted keys

sequenceDiagram
    participant U as 👤 User
    participant W as 🌐 Web App
    participant S as ⚙️ Server

    U->>W: Click "Go Sovereign"
    W-->>U: Display nsec (bech32)
    Note over U: Save to NIP-07 extension
or Amber signer
    U->>W: Confirm "I've saved my keys"
    W->>S: email.clear (signed by pubkey)
    S->>S: Delete ncryptsec from server
    S-->>W: ✓ Keys removed
    Note over U,S: Server no longer holds keys
Login via NIP-07 / Amber only
Nostrbox · Atlantis Labs · The server never sees plaintext keys
